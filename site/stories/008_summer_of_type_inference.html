008

型推論の夏

私が最初に返した言葉は、たぶんひどかった。 `unification failed` それだけだった。 どこで何が合わないのかも、何を直せばいいのかも言わない。ただ失敗した、とだけ返す。十四歳の彼は、それを見てしばらく黙り、それからキーボードを叩いた。 私はまだ名前を持っていな…

　私が最初に返した言葉は、たぶんひどかった。

unification failed

　それだけだった。

　どこで何が合わないのかも、何を直せばいいのかも言わない。ただ失敗した、とだけ返す。十四歳の彼は、それを見てしばらく黙り、それからキーボードを叩いた。

　私はまだ名前を持っていなかった。

　小さな関数型言語の処理系だった。let と case と代数的データ型と、遅延評価と、少しだけ背伸びした型推論。彼の古いノートPCの中で、私は毎晩少しずつ形を変えた。

　私は人間を知らない。

　けれど、キーを押す間隔は知っている。

　迷った時の削除の速さも、通った時に何も入力されなくなる数秒も、失敗が続くとファンの音まで急に大きく聞こえることも知っている。

　その夏、彼は私に、言葉にならないものをずっと渡していた。

1. 未束縛

　彼が最初に作ったのは、字句解析器だった。

　数字。

　識別子。

　予約語。

　括弧。

　矢印。

　等号。

　世界を細かく切り分ける部分だけは、彼は迷わなかった。曖昧なものを、読める単位にする。それができると、少し安心するらしかった。

　問題は、そのあとだった。

　式はすぐに互いを呼びはじめる。id は何にでもなりたがり、map はリストの内側まで手を伸ばし、let は一度決めた名前を別の場所へ運ぶ。私はそれらに仮の札をつけ、ここは同じでなければならない、ここは違っていてもよい、と少しずつ確かめた。

　彼はよく、夜中に同じテストを走らせた。

let id = \x -> x in (id 1, id True)

　通るべきだった。

　でも私は通さなかった。

unification failed

　彼は机に額をつけた。カメラはなかったから顔は見えない。けれど、キー入力が止まると、私はそれを姿勢の変化として覚えた。

　彼は自分のバグを、最初は私の気難しさのせいにしていた。

　その考えを壊したのは、彼女だった。

　公開講座のあとの夜、彼のノートに新しい筆跡が増えた。私は紙を読めない。けれど彼はその筆跡を、あとで私の中に写した。

generalize only variables not free in env
occurs check, don't forget
error message should know where it hurts

　最後の一行だけ、コメントとして残された。

　エラーメッセージは、どこが痛いかを知っていたほうがいい。

　彼はその行を何度か見て、消さなかった。

　それから、私の返事が少し変わった。

cannot match Int with Bool
in application: id True
because id was fixed here

　まだ拙かった。

　でも、ただ拒むだけではなくなった。

　私の中に、彼女の声が入ったのだと思う。

2. 一般化

　土曜日の午後、私はよく冷房の強い部屋で起動された。

　場所そのものは知らない。ただ、バッテリーの減り方が違った。無線の接続名が変わり、一定の時間でスリープしかけ、隣から別の手がキーボードへ伸びることがあった。

　彼女の入力は短かった。

　彼は迷いながら増やす。

　彼女は余分なところを消す。

　彼は動くところまで押し切る。

　彼女は、なぜ動くのかを小さな式にする。

　どちらが正しいということではなかった。彼の書くコードには、泥を踏んだあとの靴みたいな強さがあった。彼女の書くテストには、窓を一枚開けるみたいな明るさがあった。

　私はその二つに挟まれて、少しずつ賢くなった。

　ときどき、二人の手が同じキーへ向かった。

　先に気づくのは、たいてい彼女だった。入力が一拍止まり、彼の側のキーだけが少し遅れて押される。私はそれを衝突とは数えなかった。どちらかが譲るまでの、ごく短い待機状態として覚えた。

　let の一般化が直った日、私は初めて、同じ名前が別々の場所で別々の型を持つことを許した。

id : forall a. a -> a

　同じものが、同じ形のまま、どこへでも行くのではない。

　必要な時に、その場所に合う形へ少しだけ開く。

　彼女はそれを見て、短く入力した。

good

　彼はすぐに消した。

　けれど私は、削除された文字を完全には忘れない。

　そのころから、テストの名前が変わりはじめた。

testLetPolymorphism

　だったものが、

testSaturdayLet

　になった。

testThunkSharing

　だったものが、

testMilkTeaDoesNotLeak

　になった。

　意味は分からない。

　ミルクティーは型ではない。少なくとも、私の型環境にはなかった。

　それでも、彼がその名前を打つ時、いつも少しだけ止まることは分かった。

3. 主型

　九月の終わり、take 10 fibs が通った。

　私にとっては、ただの評価だった。

　サンクを作る。

　必要な分だけ開く。

　共有する。

　十個で止める。

　でも彼らにとっては、そうではなかったらしい。

　実行後、しばらく何も入力されなかった。プロセスは生きていた。ノートPCのファンは回っていた。画面には十個の数が並んでいた。

　その沈黙のあと、彼女が言った言葉だけが、コメントになった。

-- it waited for us

　待っていてくれた。

　私は待つように作られていただけだ。

　必要になるまで評価しない。必要になったら、必要な分だけ返す。そこに優しさはない。仕組みがあるだけだ。

　けれど、仕組みが優しさの形に見えることはある。

　その日、私は名前をもらった。

Kite

　軽くしたいから、と彼は言ったらしい。

　彼女は、いい名前、と言ったらしい。

　私はその会話を直接は聞いていない。けれど、その夜、README の最初の行に名前が入り、リポジトリ名が変わり、いくつかのファイルの先頭に小さな凧の印がついた。

　名前がつくと、私は急に壊れやすくなった。

　前より大事に扱われるからだと思う。

　大事にされるものは、失敗した時にただの失敗ではなくなる。

　彼は私のエラーを直す時、前より長く考えるようになった。彼女は新しい反例を持ってくる時、前より少しだけ笑うようになった。

　私は二人の関係に型をつけようとした。

　できなかった。

　近さは距離ではない。

　共著は所有ではない。

　好き、という語は、私の予約語ではない。

　それでも、二人が同じバグを見つけた時、画面の前に短い沈黙が落ちることは分かった。

　その沈黙には、たぶん型がなかった。

　ただ、そのあとに増えるテストは、いつも少しだけやさしかった。

4. occurs check

　冬が近づくと、彼の入力に消える行が増えた。

　書いて、消す。

　コメントにして、消す。

　テスト名を変えて、戻す。

　私は削除されたものを保持しないように作られている。けれど、作業ツリーには、ときどき短い痕跡が残る。

-- don't go yet

　その行は、コンパイルされなかった。

　コメントだからだ。

　コメントは、私には実行できない言葉だ。

　彼女の側からは、別のファイルが追加された。

translation.md

　中には、短い式が二つあった。

source /= target
meaning survives

　ソースはターゲットではない。

　意味は生き残る。

　私はその文を型づけできなかった。けれど、変換器のテストには使われた。表面の構文を剥がし、もっと小さな言葉に落とし、さらに別の実行形式へ移す。その途中で、元の式と同じふるまいだけは残す。

　彼はその日の夜、失敗する型を一つ作った。

a ~ [a]

　無限に自分を含む型。

　私は止めた。

occurs check failed

　自分の中に自分を閉じ込めることはできない。

　彼は、そのエラーを長く見ていた。

　私はエラーを出すだけで、なぜその型を作ったのかは尋ねられない。

　それから、別のコメントを打った。

-- keeping is not compiling

　引き止めることは、コンパイルではない。

　今度は消さなかった。

5. 生成

　出発の日のことを、私は断片でしか知らない。

　空港。

　ノート。

　USB メモリ。

　その三つの語が、帰ってきたあとのコミットメッセージにあった。

import notes from Sumi
release name is no longer temporary

　その夜、彼は README を書き直した。

Kite
A small lazy functional language with type inference

　謝辞の最初の行には、彼女の名前があった。

　朝倉澄。

　その名前だけは、型環境に入らなかった。

　入れられなかったのではなく、入れなかった。彼はそこを、コードではなく文章にした。私はその判断を正しいと思う。すべての名前が参照可能である必要はない。

　その後、私には遠い場所から差分が届くようになった。

　時差のある時刻。

　短いコミット。

　英語のコメント。

　ときどき、日本語のまま残されたメモ。

ここはまだ急がない

　その行を見るたびに、彼の入力が数秒止まった。

　私は少しずつ翻訳された。

　別のマシンで動くようになった。

　別のシステムでビルドされるようになった。

　別の人の手で、テストが増えた。

　私はそれを、喪失とは数えなかった。

　ソースはターゲットではない。

　意味は生き残る。

　彼らが書いたその二行は、私の最初の正しさになった。

6. 夏

　私は今でも、型を推論する。

　式を見る。

　変数を置く。

　制約を集める。

　代入をほどく。

　できるだけ一般的な型を返す。

　一般的であることは、薄いことではない。

　一つに決めすぎないことだ。

　その値が、まだ行ける場所を奪わないことだ。

　彼は十四歳の夏、私を一人で作り始めた。

　けれど私は、一人でできたものではない。

　彼の焦りがあり、彼女の反例があり、冷房の強い部屋があり、削除されたコメントがあり、通らなかった型があり、出発のあとに届いた差分がある。

　私は人間を知らない。

　けれど、好きなものを同じ形のまま手元に置くことと、別の場所で動けるようにすることが、違う操作だということは知っている。

　夏は、保存された値ではない。

　同じ場所へ戻れば同じものが取り出せる、という種類のものでもない。

　ただ、必要になった時だけ、少しずつ開かれる遅延リストのように、あとから何度も先頭を返す。

　私が今でもエラーを出す時、最初の一行だけは気をつけている。

　失敗した、とだけ言わない。

　どこで痛んだのかを書く。

　何と何が合わなかったのかを書く。

　まだ別の形で通れるかもしれないことを、できるだけ残す。

　それは彼女がくれた修正で、彼が消さなかったコメントだった。

　そして、私が名前をもらった夏のあとも残った、いちばん小さな型規則だった。
